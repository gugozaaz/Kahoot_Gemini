Kahoot! System Analysis

การวิเคราะห์โครงสร้างการออกแบบซอฟต์แวร์ด้วยเฟรมเวิร์ก Brief C.L.E.A.R.

C Customer [คนใช้มีใครบ้าง]

ผู้ใช้งานในระบบ Kahoot แบ่งออกเป็น 2 กลุ่มหลักที่มีบทบาทแตกต่างกันอย่างชัดเจน:

  • Host / Creator (ผู้สร้างและผู้คุมเกม): ได้แก่ คุณครู, อาจารย์, วิทยากร, ผู้จัดอบรม, แอดมินองค์กร หรือผู้ใช้ทั่วไปที่ต้องการสร้างกิจกรรม
  • Player / Participant (ผู้เล่น): ได้แก่ นักเรียน, นักศึกษา, พนักงาน, หรือผู้เข้าร่วมกิจกรรมทั่วไป

L Looking For [เขามองหาอะไร / เป้าหมายคืออะไร]

Host (ผู้สร้าง) มองหา:

  • เครื่องมือที่ใช้งานง่าย รวดเร็ว ในการสร้างสื่อการสอนหรือแบบทดสอบ
  • วิธีการดึงดูดความสนใจ (Engagement) และทำให้ผู้เข้าร่วมมีส่วนร่วมสูงสุด
  • ข้อมูลประเมินผลแบบเรียลไทม์ (Real-time Analytics) เพื่อดูว่าผู้เรียนเข้าใจเนื้อหาหรือไม่

Player (ผู้เล่น) มองหา:

  • ประสบการณ์การเรียนรู้หรือการประเมินผลที่สนุกสนาน ไม่ตึงเครียด (Gamification)
  • ความท้าทายและการแข่งขัน (ผ่านระบบการนับคะแนนและจัดอันดับ)
  • ความง่ายในการเข้าถึง (ไม่ต้องสมัครสมาชิก ไม่ต้องโหลดแอป แค่มี PIN ก็เล่นได้เลย)

E Experience [เขาเจอ/ทำอะไรในระบบบ้าง]

User Journey ของ Host:

  • สมัครสมาชิก / ล็อกอิน
  • สร้างชุดคำถาม (กำหนดโจทย์, ใส่รูป/วิดีโอ, กำหนดตัวเลือก, ระบุข้อที่ถูก, ตั้งเวลา)
  • กดเปิดห้องเกม (Host Game) ระบบจะสร้างหน้าจอ Lobby พร้อม Game PIN
  • ควบคุมจังหวะของเกม (กดเริ่มเกม, กดแสดงผลคะแนน, กดเปลี่ยนข้อ)
  • เมื่อจบเกม ดูสรุปผลผู้ชนะ และดาวน์โหลด Report สถิติการตอบของทุกคน

User Journey ของ Player:

  • เข้าเว็บไซต์ kahoot.it (ผ่านมือถือ/แท็บเล็ต) หรือเปิดแอปพลิเคชัน
  • กรอก Game PIN ที่เห็นบนหน้าจอของ Host และตั้งชื่อเล่น (Nickname) เพื่อเข้าห้อง
  • มองหน้าจอของ Host เพื่ออ่านคำถามและตัวเลือก
  • มองหน้าจอตัวเอง แล้วกดปุ่มสี/สัญลักษณ์ที่ตรงกับคำตอบที่ถูกต้องให้เร็วที่สุด
  • ดูคะแนนของตัวเองและลุ้นอันดับ (Leaderboard) ระหว่างข้อและตอนจบเกม

A Attributes Data [ข้อมูลสำคัญที่ระบบต้องจัดเก็บ]

ในการออกแบบฐานข้อมูลของ Kahoot ระบบจำเป็นต้องเก็บข้อมูล (Entities) หลักๆ ดังนี้:

  • User Data (ข้อมูลผู้สร้าง): User_ID, Name, Email, Password_Hash, Subscription_Plan
  • Kahoot/Quiz Data (ข้อมูลชุดคำถาม): Quiz_ID, User_ID, Title, Description, Cover_Image, Visibility
  • Question & Choice Data (ข้อมูลคำถาม): Question_ID, Quiz_ID, Question_Type, Question_Text, Media_URL, Time_Limit, Choice_ID, Choice_Text, Is_Correct
  • Session/Room Data (ข้อมูลห้องเกม): Session_ID, Quiz_ID, Host_User_ID, Game_PIN, Status, Created_At
  • Player Data (ข้อมูลผู้เล่น): Player_ID, Session_ID, Nickname, Total_Score, Current_Rank
  • Interaction/Answer Data (ข้อมูลการตอบ): Answer_ID, Player_ID, Question_ID, Selected_Choice_ID, Response_Time_Ms, Points_Earned

R Rules [กฎ / เงื่อนไข / ข้อยกเว้นของระบบ]

  • Authentication Rule: ผู้เล่น (Player) ไม่จำเป็นต้องมีบัญชีผู้ใช้ อาศัยแค่ Game PIN เพื่อเข้าห้องชั่วคราว แต่ผู้สร้าง (Host) ต้องล็อกอินเสมอ
  • Access Rule: Game PIN จะต้องตรงกับ Session ที่สถานะเป็น Active เท่านั้น หากห้องปิดแล้ว PIN นั้นจะใช้ไม่ได้
  • Scoring Rule (กฎการให้คะแนน): ต้องตอบให้ "ถูก" และ "เร็ว" หากตอบผิดหรือหมดเวลาจะได้ 0 คะแนน
  • Moderation Rule: Host มีสิทธิ์คลิก "เตะ" (Kick) ผู้เล่นที่ตั้งชื่อไม่เหมาะสมออกจาก Lobby ได้
  • UI/UX Rule: โดยค่าเริ่มต้น หน้าจอของ Player จะไม่แสดงข้อความคำถามและตัวเลือก จะแสดงเพียงปุ่มสี เพื่อบีบให้ผู้เล่นต้องเงยหน้ามองหน้าจอส่วนรวม (หน้าจอ Host)
  • Limitation Rule: ผู้เล่นสูงสุดต่อห้อง หรือประเภทคำถามพิเศษ จะถูกจำกัดการใช้งานตามระดับแพ็กเกจ (Subscription Plan) ของ Host
